# Корпоративный сайт ТОО «АВС-Лимитед»

Профессиональный многостраничный корпоративный сайт строительной компании
**ТОО «АВС-Лимитед»** (дорожное и промышленное строительство, г. Атырау, Казахстан)
с полноценной **системой управления содержимым (CMS)** на Flask + SQLite.

Весь контент сайта (тексты, изображения, проекты, техника, благодарности,
контакты, цвета бренда) редактируется через защищённую админ-панель — без правки кода.

---

## Возможности

**Публичная часть** (адаптивный дизайн, navy + красный фирменный стиль):

| Страница | Содержимое |
|----------|-----------|
| Главная | Hero, показатели, о компании, преимущества, услуги, производство, проекты, лицензии, CTA |
| О компании | История, принципы, показатели, конкурентные преимущества, лицензии и стандарты |
| Виды деятельности | 5 направлений с подпунктами |
| Производство | Производственные базы, заводы, БСУ, карьер, активы |
| Техника | Полный парк машин и оборудования (таблица) |
| Проекты | Портфолио с фильтром по категориям |
| Актуальные задачи | Текущие объекты компании |
| Расширяем границы | Мегапроекты и компетенции |
| Благодарности | Галерея писем с лайтбоксом |
| Контакты | Контактные данные, **банковские реквизиты (БИН, ИИК, банк)**, карта, форма заявки |

**Админ-панель** (`/admin`):

- Дашборд со статистикой
- Редактирование **страниц** (герой, тексты, SEO, меню)
- CRUD: показатели, преимущества, виды деятельности, лицензии,
  производственные активы, парк техники, проекты, актуальные задачи,
  «расширяем границы», благодарности
- **Настройки сайта**: бренд, логотип, контакты, реквизиты (БИН/банк), тексты блоков, цвета темы
- Заявки с формы обратной связи
- Загрузка изображений, переключение видимости, смена пароля

## Запуск

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Наполнить базу контентом из презентации (создаёт abc.db)
python seed.py

# 3. Запустить сайт
python app.py
```

- Сайт: <http://127.0.0.1:5000>
- Админ-панель: <http://127.0.0.1:5000/admin>

**Вход в админку по умолчанию:** логин `admin`, пароль `admin123`
(смените пароль в разделе «Безопасность» после первого входа).

## Деплой на Railway

Проект готов к развёртыванию на [Railway](https://railway.app) «из коробки»
(production-сервер **gunicorn**, авто-инициализация БД, поддержка тома для данных).

### Шаги

1. **Залейте код в GitHub** (см. раздел ниже) и в Railway → *New Project → Deploy from GitHub repo*.
   Railway сам определит Python, установит зависимости и запустит команду из `Procfile` / `railway.json`.
2. **Переменные** (Railway → *Variables*):
   | Переменная | Значение |
   |-----------|----------|
   | `ABS_SECRET_KEY` | длинная случайная строка (`python -c "import secrets;print(secrets.token_hex(32))"`) |
   | `DATA_DIR` | `/data` (путь тома, см. шаг 3) |
   | `ABS_ADMIN_PASSWORD` | пароль администратора при первом запуске (необязательно) |
3. **Том для постоянных данных** (важно для SQLite): Railway → *New → Volume*, точка монтирования `/data`.
   Тогда база `abc.db`, загруженные изображения и заявки **переживут редеплой**.
   Без тома данные сбрасываются к исходному наполнению при каждом деплое.
4. Railway выдаст домен — откройте его. БД создастся и наполнится автоматически при первом старте.

Healthcheck (`/`), авто-рестарт и порт (`$PORT`) уже настроены в `railway.json`.

> **PostgreSQL вместо SQLite (опционально).** Добавьте плагин *PostgreSQL* — Railway задаст
> `DATABASE_URL`, и приложение использует его автоматически. Допишите `psycopg2-binary` в `requirements.txt`.

### Локальный запуск production-сервера

```bash
DATA_DIR=. ABS_SECRET_KEY=dev python init_db.py
gunicorn app:app --bind 0.0.0.0:8000 --workers 1 --threads 8
```

## Стек и структура

- **Flask 3** + **Flask-SQLAlchemy** + **SQLite** (`abc.db`)
- Серверный рендеринг Jinja2, без сборки фронтенда
- Иконки — инлайновые SVG, шрифты — Montserrat + Inter

```

├── app.py            # приложение: публичные + админ-маршруты, обобщённый CRUD
├── models.py         # модели данных (вся CMS-схема)
├── seed.py           # наполнение БД содержимым презентации
├── init_db.py        # идемпотентная инициализация БД при деплое
├── requirements.txt  # зависимости (Flask, SQLAlchemy, gunicorn)

├── railway.json      # конфигурация деплоя Railway
├── .python-version   # версия Python для сборки
├── .env.example      # пример переменных окружения
├── static/
│   ├── css/style.css # дизайн-система сайта
│   ├── css/admin.css # дизайн-система админки
│   ├── js/main.js    # меню, анимации, счётчики, лайтбокс, фильтр

    ├── base.html, _macros.html, index.html, about.html, ...

```

## Безопасность

Встроено «из коробки»:

- **CSRF-защита** всех POST-форм (токен в сессии, проверка `before_request`).
- Защита от **open redirect** в форме входа.
- Изменяющие действия (toggle/delete/logout) — только методом **POST** + CSRF.
- **Анти-брутфорс** входа: блокировка по IP после 5 неудачных попыток.
- **Санитизация HTML-полей** при сохранении (защита от хранимого XSS).
- **Валидация** цветов темы (только hex) и e-mail в форме заявки.
- Загрузка изображений ограничена расширениями `png/jpg/jpeg/gif/webp`
  (SVG исключён — потенциальный XSS), лимит 16 МБ, имена через `uuid`.
- Cookie сессии: `HttpOnly`, `SameSite=Lax` (и `Secure` при `ABS_HTTPS=1`).
- Контроль публикации страниц (`is_published`) на публичных маршрутах.
- Устойчивые страницы ошибок (404/413/500) с откатом «грязной» сессии БД.
- Пароли — хеш (Werkzeug), все админ-маршруты под `login_required`.

### Переменные окружения

| Переменная | Назначение | По умолчанию |
|-----------|-----------|--------------|
| `ABS_SECRET_KEY` | Ключ подписи сессий. **Обязательно в проде.** | случайный на запуск |
| `FLASK_DEBUG` | `1` — включить отладчик (только локально) | `0` (выкл.) |
| `ABS_HTTPS` | `1` — за HTTPS-прокси (cookie `Secure`) | `0` |

Перед публикацией в интернете:

1. Смените пароль администратора (раздел «Безопасность» в админке).
2. Задайте `ABS_SECRET_KEY` (например `python -c "import secrets;print(secrets.token_hex(32))"`).
3. Запускайте за production-сервером (gunicorn/uWSGI + nginx), без `FLASK_DEBUG`.
